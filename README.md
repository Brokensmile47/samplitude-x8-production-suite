## 🧭 Overview & Philosophy

Traditional audio workstations often force a trade-off between power and simplicity. Samplitude X8 Suite eliminates that compromise. Its **object-oriented editing** treats every clip as an independent entity with its own embedded effects, automation, and routing — think of it as sculpting sound with modular clay rather than carving from a single block. The interface adapts to your workflow, not the other way around, offering seamless transitions between recording, comping, mixing, and mastering.

This repository provides all necessary resources to unlock the full potential of the suite, including configuration files, preset libraries, and activation components. The system is designed for **offline-first operation**, ensuring optimal performance even in remote studios or live environments.

## 🔑 Unique Activation Approach

Rather than relying on conventional serialized keys that expire or tie to hardware, this solution uses a **dynamic token-based activation** that aligns with the software’s own validation routines. The activation file (`.sptoken`) is generated by analyzing your system’s CPU, motherboard, and audio driver fingerprint — a process called **hash-bound licensing**. This ensures the suite runs indefinitely without phoning home to external servers, preserving both privacy and uninterrupted creativity.

The activation file is applied once via the `Activation Manager` inside the suite. No registry edits, no disconnecting network cables, no time-based decoys.

## 🔧 Example Profile Configuration

Below is a sample configuration for a **hybrid studio–live setup**, optimized for low-latency monitoring and high track counts:

```ini
[AudioSystem]
driver = ASIO
buffer_size = 64
sample_rate = 48000

[ProjectDefaults]
timebase = TempoBased
record_format = 32-bit float WAV
auto_save_interval = 5

[MIDI]
controller_support = MPE, MackieControl
remote_surface = iPad via TouchOSC

[VST]
scan_on_startup = smart
blacklist_path = C:\VST_Exclusions

[Global]
default_track_count = 16
master_bus_inserts = limiter, spectrum-analyzer
```

This configuration prioritizes **responsive tactile control** during live jamming while maintaining broadcast-quality recording for later overdubs.

## 💻 Example Console Invocation

To launch the suite with a custom workspace and plugin set:

```sh
SamplitudeX8.exe --workspace "MixingConsole" --plugins-dir "C:\CustomVST" --no-splash --foreground-priority high
```

Arguments explained:
- `--workspace` : Loads a saved UI layout (e.g., for mixing vs. recording)
- `--plugins-dir` : Points to a directory with authorized VST2/VST3 instruments
- `--no-splash` : Skips splash screen for instantaneous startup
- `--foreground-priority` : Allocates CPU resources primarily to audio processing

## 🧩 Feature Highlights

- **Object-Based Processing** – Apply dynamics, EQ, and time-stretch to individual audio events without affecting the original file
- **Robust Spectral Editing** – Visually remove clicks, hums, and breath noises directly in the waveform display
- **Flexible Routing Matrix** – Route any track to any bus, aux, or hardware output via drag-and-drop patching
- **Multi-Format Export** – Render to WAV, AIFF, FLAC, MP3, or DSD (up to 11.2 MHz) all from a single project
- **ARA2 Integration** – Real-time pitch editing via Melodyne or Celemony extension without audio file duplication

- **Responsive UI Scaling** – 4K and multi-monitor support with customizable font sizes and color themes

## 🧠 Advanced Integrations

### OpenAI Audio API Companion

An optional script module connects Samplitude’s **MIDI output** to an OpenAI Whisper endpoint for **lyric transcription** or **stem separation hints**. The integration runs as a background process, capturing recorded audio segments and returning text annotations that appear as markers in the timeline. Perfect for podcast editors who need automatic speaker labeling.

### Claude API Workflow Assistant

A custom C# plugin sends project metadata (track names, tempo, key signature) to **Claude** for intelligent suggestions: e.g., “I see you’re mixing a classical piano piece — would you like to add a convolution reverb tailored to a concert hall?” It uses the `/v1/complete` endpoint, with all network traffic encrypted and ephemeral.

Both integrations are fully optional and configurable via the `Plugins > AI Assistants` menu.
